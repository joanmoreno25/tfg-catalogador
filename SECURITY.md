# Security Policy

At AdVision, we take the security of our systems, our data, and our users very seriously. We are committed to ensuring our application is secure and we appreciate the efforts of the security community in helping us maintain a high standard of protection.

## Supported Versions

We currently provide security updates and patches only for the latest major version of the software. Please ensure you are reviewing the most recent codebase in the `main` branch.

| Version | Supported          |
| ------- | ------------------ |
| `main`  | :white_check_mark: |
| `< 1.0` | :x:                |

## Reporting a Vulnerability

If you believe you have discovered a potential security vulnerability in this project, **please do not disclose it publicly** via GitHub Issues, public pull requests, or social media.

Instead, we ask that you practice Responsible Disclosure by following these steps:

1. **Report the issue privately:** Send an email directly to **joan.moreno.martin@gmail.com**.
2. **Provide detailed information:** Please include a clear description of the vulnerability, the potential impact, and step-by-step instructions or a Proof of Concept (PoC) to reproduce the issue.
3. **Wait for our response:** We will acknowledge receipt of your vulnerability report within 48 hours and strive to send you regular updates about our progress.

## Scope of the Policy

This policy covers the core repository and its direct integrations. Third-party dependencies and external cloud infrastructure (e.g., AWS S3, Firebase) are subject to the security policies of their respective vendors.

Thank you for helping keep AdVision safe for everyone.
